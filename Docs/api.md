# Scanner API Implementation Guide

This document outlines the API protocol implemented by the DiscCatalog macOS client. The client uses a **Store-and-Forward** architecture, meaning it generates payloads offline (with client-generated IDs) and uploads them sequentially.

## Authentication

All requests are authenticated using an API Key.

*   **Header**: `x-api-key`
*   **Value**: The key configured in the macOS App Settings.
*   **Expected Response**: `200 OK` (401/403 will pause the uploader).

## Endpoints

The client sends data to the configured **Endpoint URL**.
*   **Local Development**: `http://localhost:3000`
*   **Production**: `https://catalog.koraybirand.com`

**Important**: Do not include port 3000 if using the HTTPS domain (Synology Reverse Proxy handles the port).

### 1. Start Session
Sent when a scan begins. This request establishes the Disk ID that will be used for all subsequent batches.

*   **URL**: `POST /api/disks/scan/start`
*   **Content-Type**: `application/json`
*   **Body**:
    ```json
    {
      "name": "MyBackupDisk",
      "total_space": 1000204886016,
      "diskId": 5821
    }
    ```
    *   `diskId`: A standard Random Int (1000-9999) generated by the client to identify this session. The server should use this ID to link subsequent batches.

### 2. Send Batch
Sent repeatedly for every directory found on the disk.

*   **URL**: `POST /api/disks/scan/batch`
*   **Content-Type**: `application/json`
*   **Body**:
    ```json
    {
      "diskId": 5821,
      "dirPath": "Photos/2024",
      "files": [
        ["Image.jpg", 102450, 1704067200000],
        ["Notes.txt", 420, 1704067250000]
      ]
    }
    ```
    *   `diskId`: Matches the ID from the Start request.
    *   `dirPath`: Relative path from the disk root. Empty string `""` for root.
    *   `files`: Array of Compacy File Objects: `[Name (String), Size (Int64), ModTime (Unix MS Int64)]`.

### 3. End Session
Sent when the scan is complete.

*   **URL**: `POST /api/disks/scan/end`
*   **Content-Type**: `application/json`
*   **Body**:
    ```json
    {
      "diskId": 5821
    }
    ```

### 4. Cancel Session
Sent if the user manually stops the scan. The server should delete any data associated with this `diskId` as the scan is incomplete.

*   **URL**: `POST /api/disks/scan/cancel`
*   **Content-Type**: `application/json`
*   **Body**:
    ```json
    {
      "diskId": 5821
    }
    ```

## Implementation Notes for Server Developer

1.  **Client-Generated IDs**: The client generates the `diskId` to support offline payload generation. The server **must accept** this ID in the `start` payload and associate it with the new Disk record.
2.  **Sequential Uploads**: The client guarantees `start` is uploaded first.
3.  **Cancellation**: If `cancel` is received, please hard-delete the Disk and all associated files/directories from the database.
4.  **Error Handling**: The client retries on 500/Network errors. It pauses on 400/401/403.
